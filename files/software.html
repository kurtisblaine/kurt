 L a n g u a g e   C r e a t i o n 

One of the most rewarding--but difficult--concepts to understand for me in computer programming is, so far, grammar creation and specifying a formal language. I have worked with LEX scanner with YAC compiler, scanning -> tokenizing -> analyzing -> generating -> and optimizing. I enjoy creating REs, DFAs, NDFA, etc. and translating it to code for a scanner, then later having it parsed by YAC compiler, constructing a formal language with proper target code--syntax semantics. I learned how programming languages are constructed--some of the greattest programs ever writen, compilers. By far, the newby programmer's best friend--also mine--a high-level compiler. A notable project I helped work on was an intermediate language that scanned in ASCII, parsed through our grammar, and translated our made up language to C code then ran by the C compiler. Our language constraints were declaring types with variables and values, executing mathematical expressions, and printing variables, floats, and strings horizontally or vertically.  A s s e m b l y   L a n g u a g e 

Assembly language programming fascinates me; I love being so close to hardware, manipulating it to exploit algorithms for optimization. I have been close to debuggers before, but assembler programming made us tight; I loved watching bits shift, stacks grow down, registers fill up, memory change, and buffers fill up, becoming real stingy with with hardware constraints--six to eight, 16-bit registers, really six. I worked with NASM, 80x86 assembler, interfacing with C and C++ with cdecl function calling conventions. Also, found assembly programming very rewarding, getting close to hardware and really learning how high-level languages translate to a low-level, converting between computer number systems--decimal, octal, hexadecimal, etc--and, of course, machine learning. A notable project I worked on was Assemblepede! You can check out my code on GitHub or, even better, play it!
